typeof

typeof 可以用来检测一个变量是不是基本数据类型（检测null 会返回object），如果检测引用类型则会返回object（检测function会返回’function’）
typeof检测null会返回object

原因是js在最初实现时，js的值是由一个标识类型的标签和实际数据值表示的，对象的类型标签是0，由于null表示空指针，所以它的类型标签是0，因此type null 会返回object。

所以，基于typeof的类型检测只能用于基本数据类型，无法判断引用类型的数据。

instanceof

instanceof的原理：判断某个构造函数的prorotype属性所指向的对象是否存在与另外一个要检测对象的原型链上。
instanceof的缺点：只适用于检验确定的变量是否属于某一种类型，但是对于未知类型的变量就不适用了。且对于基本类型就不生效（返回false），而且不能检测null和undefined（报错）
