let、const和var的区别

在JS中有三个变量可以声明变量：var、let和const，var在所有JS版本中都可以使用，let和const在ES6之后才出现的。

var

var声明的范围是函数作用域
var声明变量，如果不初始化，默认会保存为undefined
如果在函数内部定义变量时，省略了var，则这个变量会变成全局变量
重复用var声明同一个变量不会报错，变量结果是最后一次的结果。
var变量存在变量提升，即在声明之前访问变量时不会报错，此时该变量为undefined。

变量提升会把变量的声明提前到作用域的顶部，但是不会提升赋值操作。

let

let声明的范围是块级作用域。
let不能重复声明变量
let声明的变量不会提升。
在全局作用域中var声明变量会成为window对象的属性，let和const声明变量不会

  • var声明的返回是函数作用域，let和const

const

const基本和let差不多
只是const声明变量必须赋值，且如果是基本数据类型后续是不能更改的

总结

var 声明的范围是函数作用域，let 和 const 声明的范围是块作用域

var 声明的变量会被提升到函数作用域的顶部，let 和 const 声明的变量不存在提升，且具有暂时性死区特征

var 允许在同一个作用域中重复声明同一个变量，let 和 const 不允许

在全局作用域中使用 var 声明的变量会成为 window 对象的属性，let 和 const 声明的变量则不会

const 的行为与 let 基本相同，唯一一个重要的区别是，使用 const 声明的变量必须进行初始化，且不能被修改
